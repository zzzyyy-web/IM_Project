# 项目对齐文档：功能清单实现检查

## 1. 项目上下文分析
- **项目名称**: IM UI
- **技术栈**: React + TypeScript + Vite + Tailwind CSS + Zustand
- **目标**: 实现一个完整的即时通讯前端，具备高度交互性和真实感（非假功能）。
- **当前状态**: 原型阶段。基础聊天 UI 已搭建，支持文本发送、简单的图片查看、Mock 数据流。大量高级功能（语音、文件、群管理、设置细节）仅为 UI 占位或未实现。

## 2. 需求理解与差距分析 (Gap Analysis)

根据 `docs/功能清单.txt`，以下是核心功能模块的现状评估：

### 2.1 基础消息功能 (P0)
| 功能点 | 需求描述 | 当前状态 | 差距/行动项 |
| :--- | :--- | :--- | :--- |
| **文本消息** | 识别链接、手机号；Emoji | 部分实现 | 需要增强文本解析（链接/手机号高亮），集成 Emoji 选择器。 |
| **图片消息** | 多选发送、原图、预览、长按转发/保存 | 部分实现 | 需要实现图片选择器（模拟）、原图选项、长按菜单（转发/保存/撤回）。 |
| **语音消息** | 长按录制(60s)、播放动画、下载进度 | **未实现** | 需要实现录音交互（按住说话）、播放器组件、模拟下载进度。 |
| **名片消息** | 发送/查看联系人 | **未实现** | 需要实现联系人选择器、名片消息卡片组件。 |
| **正在输入** | 单聊/群聊输入状态提示 | **未实现** | 需要在 Store 中添加输入状态管理，模拟输入事件。 |
| **消息操作** | 撤回、转发(合并/逐条)、复制、收藏、删除 | 部分实现 | 撤回逻辑简单；转发/收藏仅 Alert；需要实现完整的转发选择器和收藏列表。 |
| **历史消息** | 下拉加载、定位 | **未实现** | 需要实现滚动加载逻辑（模拟分页）、搜索定位。 |
| **常用事件** | @用户、重发失败消息、拍一拍(头像) | **未实现** | 需要实现 @ 提及功能、消息发送状态重试逻辑。 |
| **文件收发** | 上传/下载/预览/转发 | **未实现** | 需要实现文件选择、文件消息卡片、模拟上传进度。 |
| **已读未读** | 状态标记、列表 | **未实现** | 需要在 Message 数据模型中完善已读状态逻辑。 |

## 3. 实施策略 (Architect & Atomize)

由于功能点众多，建议分批次迭代：

1.  **Phase 1: 消息核心增强**
    - 完善消息类型（语音、名片、文件）。
    - 完善消息操作（转发、撤回、多选）。
    - 实现输入状态和 Emoji。
2.  **Phase 2: 会话与群组管理**
    - 完善群设置、成员管理。
    - 完善联系人添加流程。
3.  **Phase 3: 全局设置与优化**
    - 主题、语言、字体。
    - 搜索功能（全局/会话内）。

## 4. 疑问与确认
- **Mock 数据**: 是否所有后端交互（上传、发送、接收）都继续使用 Mock 数据？（假设：是，保持纯前端交互）
- **UI 库**: 是否继续沿用 Shadcn UI + Tailwind？（假设：是）
- **图标库**: Lucide React 是否满足所有需求？（假设：是，不够时寻找替代）

## 5. 结论
当前项目距离“功能清单完全实现”还有较大差距。主要工作量在于**交互细节的实现**（如长按菜单、选择器弹窗、录音交互）和**缺失组件的开发**。
